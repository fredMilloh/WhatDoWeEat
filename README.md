## Introduction

Application permettant d'obtenir des recettes en fonction de l'inventaire de son frigo. L'utilisateur peut sauvegarder ses recettes préférées, en appuyant simplement sur un bouton. Il les retrouvent à n'importe quel moment dans l'onglet *Favorite*. Toute l'application est en anglais.

English version :

An application that allows users to obtain recipes based on the inventory in their fridge. Users can save their favourite recipes at the touch of a button. They can find them at any time in the *Favourite* tab.

## Features

Saisie des ingrédients :
 - Tant que la liste est vide, l'utilisateur ne peut pas lancer de recherche.
 - L'utilisateur saisi un par un les ingrédients, ou ensemble en les séparant par une virgule.
 - La saisie de plusieurs virgules ou espaces est acceptées.
 - La saisie d'un chiffre ou d'un caractère autre que lettre et virgule, engendre un message d'erreur au moment de l'appui sur le bouton *Add* 
 - Quand l'utilisateur ajoute des ingrédients, ils sont affichés en liste.
 - Les ingrédients de la liste peuvent être retirés séparément.
 - La liste peut être effacée avec le bouton *Clear*

Le résultat des recettes :
 - Les recettes correspondantes aux ingrédients saisis sont affichées en liste.
 - En sélectionnant une recette, plus de détails s'affichent dans une nouvelle vue.
 - Un bouton permet d'afficher la page web de la recette, pour avoir les consignes de préparations.

Les favoris :
 - Les recettes sauvegardées s'affichent en liste, dans l'onglet *Favorite*.
 - Si la liste est vide, un message informe l'utilisateur de la procédure à suivre pour ajouter une recette dans les favoris.
 - Dans la vue détaillée, un bouton évidé en haut à droite, permet de placer la recette dans les favoris.
 - Quand le bouton est teinté, la recette peut être retirée des favoris, avec un nouvel appui sur ce bouton.
 - Un glissé vers la gauche permet également de retirer une recette.



English version :

Entering ingredients :
 - As long as the list is empty, the user cannot launch a search.
 - The user enters the ingredients one by one, or together, separated by a comma.
 - The entry of several commas or spaces is accepted.
 - Entering a number or a character other than a letter and a comma will result in an error message when the *Add* button is pressed 
 - When the user adds ingredients, they are displayed in a list.
 - Ingredients in the list can be removed separately.
 - The list can be cleared with the *Clear* button

The result of the recipes :
 - The recipes corresponding to the entered ingredients are displayed in the list.
 - By selecting a recipe, more details are displayed in a new view.
 - A button displays the recipe's web page, to get the preparation instructions.

Favorites :
 - The saved recipes are displayed in a list in the *Favorite* tab.
 - If the list is empty, a message informs the user of the procedure to follow to add a recipe to the favorites.
 - In the detail view, a recessed button in the upper right-hand corner allows the recipe to be placed in the favorites.
 - When the button is tinted, the recipe can be removed from the favorites by pressing the button again.
 - A left drag also allows to remove a recipe.

## How To Use

Depuis votre terminal, clonez ce dépôt avec la commande git suivante :

From your terminal, clone this repository with the following git command :

```
$ git clone https://github.com/fredMilloh/WhatDoWeEat
```
**Alamofire**

La librairie Alamofire (https://cocoapods.org/pods/Alamofire) est utilisée. 

The Alamofire library (https://cocoapods.org/pods/Alamofire) is used.

Avec cocoapods, une fois le projet cloné, dans le terminal, positionnez-vous dans le répertoire du projet, et lancer `pod install`.

With cocoapods, once the project is cloned, in the terminal, go to the project directory, and run `pod install`.
